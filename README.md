# SWGOH — Планировщик ТБ Rise of the Empire

Статический сайт для планирования Территориальной Битвы **Rise of the Empire** в Star Wars: Galaxy of Heroes.

**Что умеет:**

- 📥 Загрузка гильдии по ссылке/ID со [swgoh.gg](https://swgoh.gg) (например `https://swgoh.gg/g/HWTOggWMSD6t06XEYD924w/`)
- 👥 Обзор гильдии: участники, ГП (общий / персонажи / флот), сортировка
- 🗺 Обзор всех 6 фаз: зоны, пороги звёзд, требования миссий, релик-требования платунов
- 📊 Планировщик: распределение игроков по зонам каждой фазы, прогресс к 1★/2★/3★,
  авторазложение по ГП, копирование плана текстом (для Discord), экспорт/импорт JSON
- ⚙️ Платуны: сколько слотов каждого юнита нужно на планете и сколько готовых юнитов
  есть в гильдии (по загруженным ростерам игроков)
- 🎯 Список специальных миссий по фазам

Все данные хранятся локально в браузере (localStorage) — сервер не нужен.

## Как пользоваться

1. Откройте сайт, вставьте ссылку на гильдию swgoh.gg → «Загрузить».
2. (Опционально) нажмите «Загрузить ростеры игроков», чтобы работала проверка платунов.
3. Вкладка «Планировщик» → выберите фазу → назначьте игроков по зонам или нажмите «Авторазложить».
4. «Скопировать план фазы» — готовый текст для Discord.

### Автозагрузка и Cloudflare

swgoh.gg закрыт Cloudflare (JS-челлендж) и не отдаёт CORS, поэтому ни прямые запросы,
ни публичные CORS-прокси, как правило, не проходят. Надёжных способа два:

1. **Мост браузера (рекомендуется)** — юзерскрипт `swgoh_bridge.user.js` для
   [Tampermonkey](https://www.tampermonkey.net/). Он делает запросы к swgoh.gg прямо из
   вашего браузера (который Cloudflare пропускает) и передаёт JSON сайту. Установка один раз:
   поставить Tampermonkey → открыть на сайте ссылку «установить скрипт-мост» → «Установить» →
   обновить страницу. После этого и гильдия, и все ростеры грузятся одной кнопкой.
2. **Ручной импорт** — открыть ссылку API в соседней вкладке, скопировать JSON и вставить
   в поле на сайте. Работает всегда, но для 50 ростеров утомительно.

## Публикация на GitHub Pages

1. Запушьте репозиторий на GitHub.
2. Settings → Pages → Source: `Deploy from a branch`, ветка `main`, папка `/ (root)`.
3. Сайт будет доступен по адресу `https://<логин>.github.io/<репозиторий>/`.

Локально: просто откройте `index.html` в браузере (сборка не нужна).

## Обновление данных по фазам

Данные по фазам/платунам/миссиям берутся из [genskaar/tb_empire](https://genskaar.github.io/tb_empire/html/main.html)
и лежат в `data/tb_data.js`. Чтобы обновить после изменений в игре:

```bash
node scripts/update_data.mjs
```

## Структура

```
index.html          — приложение (одна страница, вкладки)
css/style.css       — стили (тёмная тема)
js/api.js           — загрузка swgoh.gg (прямой fetch + каскад CORS-прокси)
js/app.js           — вся логика UI и планирования
data/tb_data.js     — датасет по фазам ТБ (генерируется скриптом)
scripts/update_data.mjs — обновление датасета из genskaar/tb_empire
test_fixture.html   — страница для ручного/автотеста с фиктивной гильдией
```
